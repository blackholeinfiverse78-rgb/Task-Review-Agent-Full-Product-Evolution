# Sample Payloads — Human Review Workflow

---

## Submission ID Format

`submission_id` is deterministically generated by the pipeline:
```
sub-{md5(title+description)[:12]}-{trace_id[-8:]}
```

Example: `sub-eb2e07e7c652-d42768ed`

---

## Sample 1 — PASS Submission

```json
{
  "task_id":          "demo-001",
  "task_title":       "E-Commerce REST API with Payment Integration",
  "task_description": "Built a complete e-commerce REST API with Stripe payment integration, user authentication using JWT, product catalog management, shopping cart functionality, order processing, and admin dashboard. Implemented comprehensive error handling, input validation, rate limiting, and security best practices. Repository includes 120+ unit tests with 92% code coverage, API documentation using Swagger/OpenAPI, Docker containerization, CI/CD pipeline configuration, and deployment guide. All endpoints are RESTful, properly versioned, and follow industry standards.",
  "submitted_by":     "Akash Kumar",
  "repository_url":   "https://github.com/akash/ecommerce-api",
  "module_id":        "task-review-agent",
  "schema_version":   "v1.0",
  "pdf_text":         "",
  "trace_id":         "trace-demo-001-pass",
  "current_task_id":  "T-GOV-001"
}
```

Expected pipeline output:
```json
{
  "trace_id":          "trace-demo-001-pass",
  "submission_id":     "sub-<hash>-001-pass",
  "evaluation_result": "PASS",
  "failure_type":      null,
  "selected_task_id":  "T-GOV-002",
  "selection_reason":  "PASS -> next_tasks[0] = T-GOV-002",
  "source":            "task_graph"
}
```

Submit endpoint response:
```json
{
  "trace_id":     "trace-demo-001-pass",
  "submission_id": "sub-<hash>-001-pass",
  "review_state": "PENDING_REVIEW",
  "status":       "QUEUED",
  "message":      "Evaluation complete. Pending human approval for final assignment."
}
```

## Approval Payloads

### Approve
```json
{
  "trace_id":      "trace-demo-001-pass",
  "submission_id": "sub-<hash>-001-pass",
  "action":        "approve"
}
```

Response:
```json
{
  "status":        "APPROVED",
  "submission_id": "sub-<hash>-001-pass",
  "assigned_task": "T-GOV-002"
}
```

Audit entry written:
```json
{
  "trace_id":      "trace-demo-001-pass",
  "submission_id": "sub-<hash>-001-pass",
  "system_task":   "T-GOV-002",
  "final_task":    "T-GOV-002",
  "action":        "approve",
  "timestamp":     "2026-05-07 17:00:08.227760"
}
```

---

### Reject
```json
{
  "trace_id":      "trace-demo-002-fail",
  "submission_id": "sub-<hash>-002-fail",
  "action":        "reject"
}
```

Response:
```json
{
  "status":        "REJECTED",
  "submission_id": "sub-<hash>-002-fail"
}
```

Audit entry written:
```json
{
  "trace_id":      "trace-demo-002-fail",
  "submission_id": "sub-<hash>-002-fail",
  "system_task":   "T-GOV-F01",
  "final_task":    "NONE",
  "action":        "reject",
  "timestamp":     "2026-05-07 17:00:15.000000"
}
```

---

### Modify
```json
{
  "trace_id":         "trace-demo-001-pass",
  "submission_id":    "sub-<hash>-001-pass",
  "action":           "modify",
  "override_task_id": "T-GOV-005"
}
```

Response:
```json
{
  "status":        "MODIFIED",
  "submission_id": "sub-<hash>-001-pass",
  "assigned_task": "T-GOV-005"
}
```

Audit entry written:
```json
{
  "trace_id":      "trace-demo-001-pass",
  "submission_id": "sub-<hash>-001-pass",
  "system_task":   "T-GOV-002",
  "final_task":    "T-GOV-005",
  "action":        "modify",
  "timestamp":     "2026-05-07 17:00:22.000000"
}
```

Note: `system_task` = original deterministic recommendation. `final_task` = human override. These differ — that divergence is the audit record.
